0. Foundations (Before Cybersecurity)

You cannot hack what you do not understand.

0.1 Hardware & OS
How Computers Work • CPU, RAM, Storage

CPU fetches → decodes → executes. RAM is volatile, storage persistent. OS kernel manages everything via system calls.

# Check CPU & memory
cat /proc/cpuinfo | head -20
free -h
DO: Draw the fetch-decode-execute cycle on paper.
DON'T: Think “magic” — everything is electrons and logic gates.
Google IT Fundamentals (free)
0.2 Filesystems & Binary
Binary, Hex, Filesystems

Everything is 0s and 1s. Hex = 4 bits per digit. NTFS, ext4, FAT32.

echo -n "Hello" | xxd
xxd -b secret.bin
Pros: Understand why buffer overflows exist.
Cons of skipping: You will never read memory dumps.
0.3 Command Line & VMs
Command Line Basics + Virtual Machines

ls, cd, grep, find, chmod. VirtualBox/VMware for isolation.

# Create VM snapshot
vboxmanage snapshot take "clean-state"
Linux Journey (free)
0.4 Containers
Docker & Containers

Lightweight virtualization. Docker images, containers, volumes, networks.

docker run -d --name kali kalilinux/kali-rolling

2. Networking (Critical for Hackers)

Core Models
OSI 7 Layers + TCP/IP

Physical → Data Link → Network → Transport → Session → Presentation → Application.

Wireshark Official Guide
DNS, HTTP/HTTPS, Routing, NAT, Firewalls, VPNs
  • DNS: nslookup, dig, zone transfer attacks
  • HTTP: headers, methods, status codes
  • Firewalls/VPN: iptables, OpenVPN, WireGuard
DO: Capture your own traffic with Wireshark daily.
DON'T: Trust “it’s encrypted” without verifying TLS.
